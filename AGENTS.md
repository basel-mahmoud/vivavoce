# Working agreement

This repo is a two-app monorepo (`apps/web`, `apps/mobile`) sharing
`packages/tokens`. Keep these conventions:

- **TypeScript strict everywhere.** No `any` without a written reason.
- **Validate at every boundary with Zod.** Never trust client input; parse it.
- **Parameterized SQL only** (Drizzle). Raw SQL must be isolated and reviewed.
- **Ownership + tenant checks** on every user-scoped query. No exceptions.
- **No secrets in code or logs.** Redact PII and audio/transcript content in logs.
- **Conventional commits**, small and traceable. Author is the repo owner only —
  no co-author or "generated by" trailers anywhere.
- **Every interactive UI element** designs default/hover/active/focus/disabled/
  loading; every async view designs loading/empty/error.
- **Respect `prefers-reduced-motion`** and accessibility from the first draft,
  not as a retrofit.

Before a commit: review the diff, drop debug/dead code, confirm no secrets, and
run the checks relevant to what changed (`web:typecheck`, `web:lint`, `web:test`,
`mobile:typecheck`).
